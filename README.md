# Poker Ledger

Track poker buy-ins, cash-outs, chips, and session stats.

**Stack:** React (Vite) + Django REST API + Postgres in production. The built frontend is served by Django as a single app.

---

## Prerequisites

| Tool | Needed for |
|------|------------|
| [Docker Desktop](https://docs.docker.com/get-docker/) | Quickest way to run locally (recommended for testing) |
| Python 3.12 + Node 20 | Active development with hot reload (`./start.sh`) |
| Git | Pushing changes |

## Run locally

Pick one approach. You do not need both.

### Option A — Docker (recommended for testing)

No Python, Node, or `.env` file required. Data is stored in a Docker volume and survives restarts.

```bash
./docker-up.sh
```

Open **http://localhost:8000**

If port 8000 is already in use:

```bash
HOST_PORT=8080 ./docker-up.sh
```

Equivalent command:

```bash
docker compose -f docker-compose.local.yml up --build
```

| Task | Command |
|------|---------|
| Stop | `docker compose -f docker-compose.local.yml down` |
| Wipe all data | `docker compose -f docker-compose.local.yml down -v` |
| View logs | `docker compose -f docker-compose.local.yml logs -f` |

Docker local mode runs without Clerk. Auth uses a built-in dev user so you can use the app immediately.

### Option B — Development (hot reload)

Use this when you are editing frontend or backend code.

**1. One-time setup**

```bash
python3.12 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cd frontend && npm install && cd ..

cp .env.example .env
```

**2. Start both servers**

```bash
./start.sh
```

| Service | URL |
|---------|-----|
| App (Vite) | http://127.0.0.1:5173 |
| API (Django) | http://127.0.0.1:8000/api/ |

Vite proxies `/api` to Django, so use the **5173** URL in the browser.

Without Clerk keys in `.env`, the backend accepts a dev bearer token automatically while `DEBUG=True`.

**3. Production-style build (optional)**

Simulates what runs in production:

```bash
./build.sh
cd backend && gunicorn config.wsgi --bind 0.0.0.0:8000
```

Open **http://127.0.0.1:8000**

## Environment variables

Copy `.env.example` to `.env` for local dev. Docker local mode sets these for you.

| Variable | Local dev | Production (Render) |
|----------|-----------|---------------------|
| `SECRET_KEY` | Any string | Auto-generated by Render |
| `DEBUG` | `True` | `False` |
| `ALLOWED_HOSTS` | `127.0.0.1,localhost` | `*` |
| `DATABASE_URL` | Unset (SQLite) | Postgres (linked in Render) |
| `CLERK_ISSUER` | Optional | Required when auth is enabled |
| `VITE_CLERK_PUBLISHABLE_KEY` | Optional | Required at **build time** when auth is enabled |

Clerk is not required for Docker local testing or for `./start.sh` with `DEBUG=True`.

## Deploy — push to `main`

Production is hosted on [Render](https://render.com). Pushing to the `main` branch triggers an automatic deploy.

### First-time setup (once per project)

1. Push this repo to GitHub.
2. In Render: **New → Blueprint** and point it at the repo. Render reads `render.yaml`, which creates:
   - A **web service** (`poker-ledger`) — runs `./build.sh`, then gunicorn
   - A **Postgres database** (`poker-ledger-db`) — linked via `DATABASE_URL`
3. In the Render dashboard, set any env vars marked `sync: false` in `render.yaml` (e.g. Clerk keys when you enable auth).
4. Note the public URL Render assigns (e.g. `https://poker-ledger.onrender.com`).

### Ship a change

```bash
# work on a branch
git checkout -b my-feature
# ... make changes, commit ...

# bring changes to main and deploy
git checkout main
git pull origin main
git merge my-feature
git push origin main
```

Render picks up the push to `main`, rebuilds the app, runs migrations, and restarts the service. Watch progress in the Render dashboard under **Events**.

### What the deploy does

`render.yaml` runs:

1. `./build.sh` — install Python deps, `npm ci && npm run build`, `collectstatic`, `migrate`
2. `gunicorn config.wsgi` — serves the API and the built React app on one port

Data in production lives in **Render Postgres** (`poker-ledger-db`), not on the web service filesystem.

### Production Docker (optional)

To run the production image locally with Postgres:

```bash
cp .env.example .env   # set SECRET_KEY
docker compose up --build
```
